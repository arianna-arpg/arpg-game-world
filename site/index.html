Why it's different

An ARPG that's systems all the way down

Most of the depth you'd build with special cases, Hollow Wake builds with data. That single decision changes how the whole game plays.

Skills are loot

Skills drop from monsters and level up by being sacrificed at fonts — not from your character level. Your build is what you find and how you socket it. Character XP buys passive points instead.

Classes are starting points, not cages

Each of the 15 classes is a starting spread of attributes and signature skills — and nothing more. Every class draws from the same shared skill and passive pools, so where you end up is entirely yours.

One engine runs everything

The player, monsters, and minions all act through a single skill pipeline. A summoner's skeletons literally run on the same skills the monsters do — because to the engine, there's no difference.

An endless, hand-seeded frontier

Two hand-authored anchors — the starting town and Wayfarer's Crossroads — open onto an effectively infinite procedural world, reshaped by optional per-run content packages and rival factions warring on shared ground.

Under the hood

Composed from data, resolved by one shared engine

Skills, supports, monsters, statuses, passives, items and zones are all plain data entries. Adding content means adding an entry under src/data/ — the engine already knows how to run it.

Every number in the game flows through the same layered, tag-scaled modifier formula, implemented once and shared by every system. Content authors compose behaviour out of tags and modifiers rather than writing new engine code — which is why the content count grows without the engine getting heavier.

It's written in TypeScript, rendered on an HTML5 Canvas 2D layer called the visual fabric, and wrapped in an Electron desktop shell with a one-click update flow.

Skills & Supports

One schema, endless combos

Actives and the support gems that reshape them share one schema. Socket a support and it warps the skill's tags, projectiles, repeats, or targeting — no special-casing. Browse every one in the Database.

Items & Affixes

A full loot economy

Base families, a wide affix gamut, uniques, and socketable vestiges with Epitaph words — rolled by one generator into a grid "tetris-bag" inventory.

Passive Tree

One vast graph

Keystones and notables, with raw attributes placed on the graph itself. Points come from XP; the tree is where classes quietly diverge — now plannable in the build tool.

AI & Factions

Enemies with agendas

Composable AI archetypes drive every monster, and rival factions hold ally/hostile relationships — so enemies fight each other on shared terrain, not just you. Meet the bestiary in the Database.

Procedural Frontier

A world with no edge

Past the two authored anchors, level generation runs over a growing matrix of tilesets, variants, formations and compositions — verified by a headless QA sweep that asserts the generation invariants.

Visual Fabric

Data-driven rendering

One flat colour becomes a full shaded look: a materials registry, baked sprites and bodies, a doodad-painter library, a dynamic light layer, and weather — new doodads are one data entry.

Co-op

Host-authoritative multiplayer

Play together locally or over WebRTC. The host owns the simulation; the same engine and save meta-layer run underneath a solo or shared session.

Death & the Vault

Stakes you can choose

Permadeath with corpse recovery is the default — but content packages like Immortal and Mercenary rewrite the rules of death. An account-level Vault carries progression across characters, whatever mode you play.

Systems vocabulary

The shared alphabet

Ten attributes, a spread of damage types, and a deep vocabulary of status effects, charge resources and procs — the primitives every skill, monster and item is spelled out of.

Built to be verified

Harnesses, not vibes

No unit-test runner — instead, the real engine runs headless and deterministic. Correctness is a type-check plus a set of self-checking harnesses you run after touching the area they cover.

npm run sim

Balance harness

The real engine, headless and seeded. Ranks every skill at equal investment, runs the skill × support no-op matrix, prints per-class power curves, and audits the item economy.

npm run perf

Performance harness

Boots the real desktop game and samples true frame telemetry per procedural zone, gating each biome against the same run's town control.

npm run genqa

Generation QA

Runs level generation across the whole authored tileset and variant matrix over many seeds, asserting determinism, reachability and the other generation invariants.

npm run smoke

Boot smoke tests

Headless Electron self-checks that boot the real game and launcher windows and assert they actually come up before you ship.

Run from source

Requires Node.js and npm. Then:
# first-time setup
npm install

# browser dev mode → localhost:5173
npm run dev

# the desktop app (Electron)
npm run game

# type-check + production build
npm run build
Controls: WASD to move, LMB/RMB + 1–6 for skills, C / B / P / M for character, skills, tree and map — all rebindable.
Early prototype

Hollow Wake is in active development. The systems are broad and working — skills, supports, items, passives, procedural worlds, AI, factions, co-op and permadeath — but the presentation is intentionally minimal for now: art is deliberate placeholder geometry rendered through the shading layer, and audio isn't in yet. Expect balance, content and class identity to keep shifting between versions.
