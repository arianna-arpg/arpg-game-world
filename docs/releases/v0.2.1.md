Steam Deck launch fix. If v0.2.0 gave you an infinite loading screen when
launched from Steam — while running the AppImage directly worked — this
release is the cure.

**Fixed**
- **Launching from Steam on Linux/Steam Deck no longer hangs.** Steam
  injects its overlay library (`gameoverlayrenderer.so`) into every game it
  starts; Chromium's child processes choked on it and the window never
  appeared. The Linux binary now ships behind a tiny shim that strips the
  overlay preload before the engine boots (other preloads pass through).
  Trade-off: the Steam overlay doesn't draw over the game.

**Changed**
- **Game Mode boots straight into the game** — no launcher window first.
  The launcher remains available from Desktop Mode (updates, factory
  reset). Opt out with `launcher.autoPlayOnGamescope: false`.

**Added**
- **Boot breadcrumbs**: every launch writes `launcher.log` into the game's
  data folder (`~/.config/Hollow Wake/` on Linux, `%AppData%\Hollow Wake\`
  on Windows) recording each boot milestone and any fatal error — so a
  silent failure names its own stall point. Disable with
  `debug.bootLog: false`.

**Updating on the Deck**: re-run the install one-liner from STEAM.md, or
download this release's AppImage and replace the old file. Saves are
untouched either way.
